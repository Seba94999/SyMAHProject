# Skill: Badge

## Responsabilidad

Renderizar indicadores visuales reutilizables para representar estados, categorías y etiquetas dentro de SYMAH.

Los badges deben comunicar información rápida y visualmente clara sin saturar la interfaz.

---

# Filosofía Visual

Los badges deben sentirse como:

- indicadores operativos
- etiquetas empresariales
- estados visuales rápidos
- señales de información contextual

Evitar:

- badges excesivamente grandes
- colores saturados
- estilos planos
- exceso de texto

---

# Objetivo

Los badges deben permitir identificar rápidamente:

- estados
- categorías
- prioridades
- actividad
- alertas
- tipos de entidad

---

# Uso

Utilizar badges para:

- estados de trabajos
- estados de empleados
- pagos
- cobros
- actividad
- prioridades
- alertas
- categorías financieras

---

# Estructura

Todo badge debe soportar:

- texto
- color semántico
- icono opcional
- estado visual
- tamaño configurable

---

# Variantes

## Success Badge

Usar para:

- activo
- completado
- aprobado
- pagado

Color:

- verde

---

## Danger Badge

Usar para:

- error
- cancelado
- deuda
- inactivo

Color:

- rojo

---

## Warning Badge

Usar para:

- pausa
- advertencia
- pendiente

Color:

- amarillo

---

## Info Badge

Usar para:

- información
- procesos
- estados neutrales

Color:

- azul

---

## Neutral Badge

Usar para:

- estados secundarios
- categorías generales

Color:

- gris

---

# Estilo Visual

## Fondo

- semántico suave
- contraste equilibrado
- integración con dark theme

---

## Tipografía

- compacta
- clara
- legible
- font-weight medio/alto

---

## Bordes

- redondeados
- suaves
- consistentes con el design system

---

# Tamaños

Debe soportar:

- small
- medium
- large opcional

---

# Iconografía

Los badges pueden incluir:

- iconos de estado
- indicadores visuales
- símbolos contextuales

---

# Comportamiento

## Hover

Opcional según contexto.

Si existe hover:

- glow suave
- mejor contraste
- transición ligera

---

# Responsive

Los badges deben:

- mantener legibilidad
- adaptarse al contenido
- evitar overflow visual

---

# Integración

Los badges deben integrarse correctamente con:

- table
- card
- modal
- dashboard
- kpi
- statusIcon

---

# UX

Los badges deben:

- comunicar rápidamente
- reducir lectura innecesaria
- mejorar escaneo visual
- reforzar estados semánticos

---

# Reglas Visuales

Los badges deben usar:

- colores semánticos
- spacing compacto
- diseño minimalista
- contraste adecuado

---

# Restricciones

Nunca:

- usar estilos inline
- usar colores excesivos
- generar badges gigantes
- romper el design system
- usar texto excesivamente largo

---

# Objetivo Final

Los badges deben funcionar como un sistema visual rápido y consistente para representar estados y contexto operativo dentro de SYMAH.
