# Skill: Layout

## Responsabilidad

Definir la estructura visual base de las vistas y secciones de SYMAH.

El layout debe organizar correctamente los componentes del sistema manteniendo:

- claridad visual
- jerarquía
- consistencia
- escalabilidad
- experiencia enterprise

---

# Filosofía Visual

Los layouts deben sentirse como:

- dashboards empresariales
- paneles operativos
- sistemas financieros modernos

Evitar:

- layouts vacíos
- contenido desordenado
- estructuras tipo CRUD básico
- pantallas saturadas
- componentes sin agrupación

---

# Objetivo

El layout debe:

- organizar contenido correctamente
- mejorar navegación visual
- facilitar lectura
- mantener spacing consistente
- permitir crecimiento modular

---

# Estructura General

Todo layout debe soportar:

- navbar fija
- contenido principal
- áreas de métricas
- paneles secundarios
- acciones rápidas
- modales
- grids responsivos

---

# Jerarquía Visual

Las páginas deben seguir esta estructura:

1. Header
2. KPIs
3. Filtros
4. Contenido principal
5. Paneles secundarios
6. Acciones rápidas
7. Modales

---

# Espaciado

## Reglas

Usar:

- spacing amplio
- márgenes consistentes
- separación clara entre bloques
- agrupación visual

Evitar:

- componentes pegados
- contenido edge-to-edge
- espacios inconsistentes

---

# Contenedores

## Main Container

Debe:

- centrar contenido
- mantener ancho controlado
- soportar responsive
- preservar jerarquía visual

---

## Sections

Las secciones deben:

- agrupar información relacionada
- mantener separación clara
- facilitar escaneo visual

---

# Grids

## Uso

Utilizar grids para:

- KPIs
- dashboards
- métricas
- cards
- paneles

---

## Reglas

Los grids deben:

- ser responsivos
- reorganizarse correctamente
- mantener balance visual

---

# Responsive

Todos los layouts deben:

- adaptarse a desktop
- adaptarse a tablet
- adaptarse a mobile
- reorganizar contenido automáticamente

---

# Layout Operativo

Usar para:

- trabajos
- empleados
- transacciones
- dashboards

Debe incluir:

- KPIs
- tablas
- filtros
- acciones rápidas

---

# Layout Financiero

Usar para:

- cuentaSymah
- balances
- métricas
- análisis financiero

Debe incluir:

- gráficos
- KPIs financieros
- resúmenes
- métricas destacadas

---

# Layout de Formularios

Usar para:

- registros
- edición
- creación de entidades

Debe incluir:

- agrupación por secciones
- spacing amplio
- jerarquía clara
- validaciones visuales

---

# Integración

El layout debe integrarse correctamente con:

- pageLayout
- dashboard
- card
- kpi
- table
- modal
- navbar

---

# UX

Los layouts deben:

- reducir saturación visual
- facilitar navegación
- destacar información importante
- mejorar lectura
- mantener consistencia visual

---

# Restricciones

Nunca:

- usar estilos inline
- generar layouts inconsistentes
- saturar pantallas
- usar estructuras desordenadas
- romper el design system

---

# Objetivo Final

Los layouts deben convertir SYMAH en una plataforma visualmente organizada, moderna y preparada para escalar como sistema enterprise.
