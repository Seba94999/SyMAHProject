# Skill: Dashboard

## Responsabilidad

Definir la estructura y comportamiento visual de los dashboards en SYMAH.

Los dashboards son el principal centro operativo y financiero del sistema.

Deben mostrar información crítica de forma clara, rápida y visualmente jerárquica.

---

# Filosofía Visual

Los dashboards deben sentirse como:

- centros de control empresariales
- paneles operativos
- sistemas financieros modernos
- plataformas de monitoreo

Evitar:

- pantallas vacías
- tablas dominantes
- layouts CRUD básicos
- métricas desordenadas
- exceso de texto

---

# Objetivo

Los dashboards deben permitir visualizar rápidamente:

- estado financiero
- actividad operativa
- métricas clave
- alertas
- progreso
- trazabilidad
- resúmenes empresariales

---

# Estructura General

Todo dashboard debe incluir:

1. Header
2. KPIs
3. Filtros
4. Contenido principal
5. Paneles secundarios
6. Acciones rápidas

---

# Header

## Debe soportar

- título principal
- subtítulo
- breadcrumbs opcionales
- acciones rápidas
- filtros rápidos

---

# KPI Section

## Uso obligatorio

Los dashboards deben priorizar KPIs visuales.

Utilizar para:

- caja
- ingresos
- gastos
- ganancias
- cobros
- empleados
- trabajos activos

---

## Reglas

- usar grids responsivos
- destacar métricas importantes
- mantener jerarquía visual

---

# Contenido Principal

## Puede incluir

- tablas
- gráficos
- actividad reciente
- timelines
- resúmenes
- estados operativos

---

## Reglas

- priorizar cards y paneles
- evitar tablas extensas como foco principal
- agrupar información relacionada

---

# Paneles Secundarios

Usar para:

- alertas
- actividad reciente
- accesos rápidos
- estados críticos
- indicadores operativos

---

# Visualización Financiera

Los dashboards financieros deben incluir:

- KPIs monetarios
- gráficos comparativos
- balances
- ingresos vs egresos
- métricas destacadas

---

# Dashboard Operativo

Usar para:

- trabajos
- empleados
- actividad diaria
- estados operativos

Debe incluir:

- progreso
- estados
- alertas
- actividad reciente

---

# Grids

## Reglas

Todos los dashboards deben usar:

- grids responsivos
- spacing consistente
- distribución balanceada

---

# Responsive

Los dashboards deben:

- reorganizar componentes automáticamente
- mantener legibilidad
- priorizar KPIs en mobile
- reducir saturación visual

---

# Integración

Los dashboards deben integrarse correctamente con:

- card
- kpi
- chart
- table
- modal
- badge
- pageLayout

---

# UX

Los dashboards deben:

- facilitar lectura rápida
- reducir esfuerzo visual
- destacar información crítica
- mejorar toma de decisiones
- transmitir sensación de control

---

# Composición Visual

Priorizar:

- cards
- KPIs
- gráficos
- indicadores visuales
- métricas destacadas
- badges semánticos

Las tablas deben complementar el dashboard, no dominarlo.

---

# Restricciones

Nunca:

- usar estilos inline
- saturar pantallas
- usar tablas como único contenido
- romper el design system
- generar layouts desorganizados

---

# Objetivo Final

Los dashboards deben convertir SYMAH en una plataforma visualmente moderna, empresarial y preparada para monitoreo operativo y financiero en tiempo real.
