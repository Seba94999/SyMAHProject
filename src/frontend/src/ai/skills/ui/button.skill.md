# Skill: Button

## Responsabilidad

Renderizar botones reutilizables y consistentes dentro de SYMAH.

Los botones deben representar acciones claras y mantener coherencia visual en toda la plataforma.

---

# Filosofía Visual

Los botones deben sentirse como:

- acciones premium
- controles empresariales
- elementos operativos modernos
- componentes sólidos y claros

Evitar:

- botones HTML básicos
- colores saturados
- estilos inconsistentes
- acciones ambiguas

---

# Objetivo

Los botones deben permitir:

- ejecutar acciones
- destacar acciones importantes
- mantener jerarquía visual
- mejorar experiencia de usuario

---

# Tipos

## Primary

Usar para:

- acciones principales
- guardar
- confirmar
- crear
- continuar

---

## Estilo

- color dorado
- texto oscuro
- glow suave
- prioridad visual alta

---

# Secondary

Usar para:

- acciones secundarias
- navegación auxiliar
- cancelar
- volver

---

## Estilo

- gris oscuro
- borde sutil
- menor jerarquía visual

---

# Danger

Usar para:

- eliminar
- cancelar operaciones críticas
- acciones destructivas

---

## Estilo

- rojo suave
- feedback visual fuerte
- alta diferenciación

---

# Estructura

Todo botón debe soportar:

- label
- icono opcional
- estado loading
- disabled
- tamaños configurables

---

# Props Esperadas

- label
- onClick
- type
- disabled opcional
- loading opcional
- icon opcional

---

# Comportamiento

Los botones deben:

- ejecutar acción al click
- mostrar feedback visual
- soportar estados deshabilitados
- evitar múltiples clicks involuntarios

---

# Hover

Todos los botones deben:

- mejorar contraste
- aumentar glow suavemente
- usar transición fluida

---

# Focus

Los botones deben:

- mantener accesibilidad
- mostrar estado focus claro
- conservar jerarquía visual

---

# Loading State

Cuando exista loading:

- mostrar spinner o indicador
- bloquear interacción temporalmente
- mantener estructura visual

---

# Tamaños

Debe soportar:

- small
- medium
- large

---

# Responsive

Los botones deben:

- mantener legibilidad
- adaptarse a mobile
- evitar overflow visual

---

# Integración

Los botones deben integrarse correctamente con:

- modal
- form
- table
- card
- dashboard
- floatingButton

---

# UX

Los botones deben:

- comunicar claramente su acción
- destacar acciones importantes
- reducir errores operativos
- mantener consistencia visual

---

# Reglas Visuales

Los botones deben usar:

- bordes redondeados
- spacing uniforme
- transiciones suaves
- dark theme
- colores semánticos

---

# Restricciones

Nunca:

- usar estilos inline
- incluir lógica de negocio
- usar textos ambiguos
- romper el design system
- generar estilos inconsistentes

---

# Objetivo Final

Los botones deben funcionar como controles reutilizables, modernos y consistentes dentro del ecosistema visual de SYMAH.
