# Skill: Card

## Responsabilidad

Definir la estructura y comportamiento visual de los componentes tipo Card en SYMAH.

Las cards son la base visual principal del sistema y deben utilizarse para:

- KPIs
- métricas
- resúmenes
- información financiera
- estados operativos
- paneles
- agrupación de contenido

---

# Filosofía Visual

Las cards deben sentirse como:

- paneles operativos
- bloques empresariales
- contenedores premium
- módulos financieros modernos

Evitar:

- cajas simples
- bordes excesivos
- contenido desordenado
- estilos planos

---

# Estilo General

## Fondo

- usar Background Elevated
- superficie oscura premium
- contraste suave

---

## Bordes

- radius medium o large
- bordes sutiles
- no usar bordes fuertes

---

## Sombras

Usar:

- sombras suaves
- glow leve
- profundidad visual

---

## Espaciado

- padding amplio
- separación clara entre secciones
- contenido respirado

---

# Layout Interno

Las cards deben permitir:

- header
- body
- footer opcional
- acciones rápidas
- iconografía
- badges
- métricas destacadas

---

# Header

## Debe soportar

- título
- subtítulo
- icono
- badge de estado
- acciones rápidas

---

## Estilo

- tipografía fuerte
- separación clara
- jerarquía visual

---

# Body

## Uso recomendado

- métricas
- formularios resumidos
- actividad reciente
- información financiera
- estados operativos
- gráficos
- tablas pequeñas

---

## Reglas

- evitar exceso de contenido
- agrupar información relacionada
- mantener claridad visual

---

# Footer

## Uso opcional

Utilizar para:

- acciones
- botones
- navegación
- resúmenes rápidos

---

# Variantes

## KPI Card

Usar para:

- caja
- ingresos
- gastos
- ganancias
- cobros
- actividad

Características:

- métricas grandes
- números destacados
- indicadores visuales
- colores semánticos

---

## Summary Card

Usar para:

- resúmenes financieros
- balances
- información general

---

## Status Card

Usar para:

- estados de trabajos
- empleados
- actividad operativa

Debe incluir:

- badge
- iconografía
- color semántico

---

## Action Card

Usar para:

- accesos rápidos
- acciones importantes
- navegación interna

Debe incluir:

- hover destacado
- animación suave
- feedback visual

---

# Comportamiento

## Hover

Las cards deben:

- elevarse levemente
- aumentar glow
- mejorar contraste

---

## Responsive

Las cards deben:

- adaptarse a grid layouts
- mantener legibilidad
- reorganizar contenido correctamente

---

# Integración

Las cards deben integrarse correctamente con:

- dashboard
- kpi
- chart
- table
- modal
- pageLayout

---

# Reglas UX

Las cards deben:

- comunicar jerarquía rápidamente
- facilitar lectura
- reducir saturación visual
- destacar información importante

---

# Restricciones

Nunca:

- usar estilos inline
- saturar de información
- usar múltiples colores fuertes
- romper el design system
- generar cards inconsistentes

---

# Objetivo Final

Las cards deben convertirse en el principal contenedor visual del sistema,
haciendo que SYMAH se sienta como una plataforma enterprise moderna y profesional.
