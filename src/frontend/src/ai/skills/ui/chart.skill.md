# Skill: Chart

## Responsabilidad

Definir la estructura visual y funcional de los gráficos en SYMAH.

Los charts deben transformar información financiera y operativa en visualizaciones claras, rápidas y fáciles de interpretar.

---

# Filosofía Visual

Los gráficos deben sentirse como:

- visualizaciones empresariales
- paneles financieros modernos
- herramientas de monitoreo operativo
- analytics dashboards

Evitar:

- gráficos saturados
- exceso de colores
- estilos genéricos
- visualizaciones difíciles de leer

---

# Objetivo

Los charts deben permitir visualizar rápidamente:

- evolución financiera
- ingresos y egresos
- ganancias
- actividad operativa
- tendencias
- comparativas
- estados

---

# Uso

Utilizar charts para:

- ingresos vs egresos
- gastos
- cobros
- balances
- evolución mensual
- trabajos activos
- productividad
- métricas operativas

---

# Tipos de Gráficos

## Bar Chart

Usar para:

- comparativas
- balances
- métricas mensuales
- ingresos/egresos

---

## Line Chart

Usar para:

- evolución temporal
- tendencias
- crecimiento
- actividad continua

---

## Pie / Donut Chart

Usar para:

- distribución de gastos
- categorías
- porcentajes

Evitar exceso de segmentos.

---

## Area Chart

Usar para:

- evolución acumulativa
- actividad financiera
- métricas históricas

---

# Estilo Visual

## Fondo

- transparente o integrado con Card
- sin ruido visual
- limpio y minimalista

---

## Colores

Usar colores semánticos:

- dorado → métricas principales
- verde → positivo
- rojo → negativo
- azul → información
- amarillo → advertencia

Evitar:

- colores excesivamente saturados
- demasiados colores simultáneos

---

# Tipografía

Los charts deben usar:

- texto claro
- tamaños legibles
- etiquetas simples
- jerarquía visual limpia

---

# Líneas y Barras

## Reglas

- bordes suaves
- animaciones sutiles
- grosor consistente
- contraste adecuado

---

# Tooltips

## Comportamiento

Los tooltips deben:

- mostrar información clara
- usar dark theme
- mantener coherencia visual
- facilitar lectura rápida

---

# Leyendas

Las leyendas deben:

- ser compactas
- usar colores semánticos
- mantener claridad visual

---

# Layout

Los charts deben:

- integrarse dentro de cards
- mantener spacing consistente
- respetar grids responsivos

---

# Responsive

Todos los charts deben:

- adaptarse a distintos tamaños
- mantener legibilidad
- reorganizar etiquetas correctamente
- evitar overflow visual

---

# Integración

Los charts deben integrarse correctamente con:

- dashboard
- kpi
- card
- pageLayout
- table

---

# UX

Los charts deben:

- facilitar lectura rápida
- simplificar análisis
- destacar tendencias importantes
- mejorar toma de decisiones
- reducir ruido visual

---

# Composición Visual

Priorizar:

- simplicidad
- claridad
- contraste
- jerarquía visual
- lectura inmediata

---

# Restricciones

Nunca:

- usar estilos inline
- usar demasiados colores
- saturar de datos
- generar gráficos complejos innecesarios
- romper el design system

---

# Objetivo Final

Los charts deben convertir los datos operativos y financieros de SYMAH en visualizaciones modernas, empresariales y fáciles de interpretar.
